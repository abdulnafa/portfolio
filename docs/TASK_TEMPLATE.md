# Task Template

Copy this block into [TASKS.md](TASKS.md) under the right section when creating a new task.
Use the next free `T-NNN` id.

```markdown
### T-NNN · <short title>
- **Priority:** High | Medium | Low
- **Status:** todo | in progress | done | blocked
- **Why:** <the problem or goal, in one or two sentences>
- **Files:** <files/paths expected to change>
- **Steps:**
  1. <step>
  2. <step>
- **Done when:** <clear, checkable acceptance criteria>
- **Verification:** <how it was tested — browser, breakpoint, link check>
```

## Guidance

- Keep one task focused on one outcome; split large efforts into multiple `T-NNN` tasks.
- Record the task **before** editing, then update its status as you go.
- On completion, move the task to the `## Done` section and note how it was verified.
- Cross-link long-form design decisions to `NEXT_CHANGES.md` when needed.
